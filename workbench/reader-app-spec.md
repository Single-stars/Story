# Story OS 移动小说阅读器产品与交互规格（研究稿）

> 状态：第一轮独立研究稿
> 日期：2026-07-16
> 适用阶段：Phase 7 Mobile Read-Only PWA，以及 Phase 8 Feedback 的前置设计
> 约束：本文只定义阅读器产品方向与技术边界，不冻结 Story OS 数据接口，也不包含应用代码。

## 1. 结论摘要

Story OS 的移动阅读器不应模仿商业阅读平台的“书城”，而应做成一个安静、可信、可恢复的私人阅读空间：

- 首页先回答“我上次读到哪里”，而不是推荐更多内容。
- 阅读页以正文为唯一视觉主角；目录、搜索、设置、分享、反馈都作为按需出现的辅助层。
- 默认采用连续纵向滚动，章末提供清晰的上一章/下一章按钮；左右点击或滑动翻页只能作为可选增强，不能成为唯一操作。
- 阅读位置必须绑定稳定章节和文本锚点，不能只保存百分比；字号、行距和屏幕宽度变化后仍应尽量回到同一句附近。
- 搜索、离线包、分享元数据和页面正文必须在构建阶段按 internal / editor / reader / public 可见性裁剪。前端隐藏字段不构成权限控制。
- v0.1 不请求位置、通讯录、相机、麦克风、通知等敏感权限。分享、剪贴板和持久离线存储只在用户明确操作时触发。
- “离线可读”与“即时撤权”存在天然冲突：内容一旦进入设备离线缓存，服务器无法在设备断网时立即删除。高敏感内容必须禁用离线，或接受并明确告知这一风险。
- 反馈是独立记录，只能进入 feedback 层；它可以关联章节与段落，但不能直接修改正文或 Canon。
- 无障碍不是收尾补丁。语义结构、焦点管理、字体缩放、对比度、触控尺寸和减少动画需要从组件设计开始落实。

推荐的 v0.1 产品范围是：访问门、书架、阅读页、目录、切章、进度恢复、排版与主题、书内搜索、受控分享、授权离线、基础无障碍。反馈入口放入 v0.2，但 v0.1 应预留锚点和状态模型。

## 2. 研究范围与判断方法

本轮选择了中文与国际成熟阅读产品的官方产品说明和帮助资料作为样本，包括 Apple Books、Amazon Kindle、Kobo、Google Play Books、微信读书、起点读书和番茄小说。它们的业务模式不同，但在长期稳定的阅读交互上高度趋同：

- Kindle、Apple Books、Kobo 和 Google Play Books 都强调字号、字体、亮度/背景、夜间阅读、滚动方式或跨设备续读。
- Kindle 明确展示章节/全书剩余时间、百分比、真实页码、书签、笔记、书内搜索和 Page Flip；Google Play Books 把章节、书签、笔记等统一放入 Contents 入口。
- Google Play Books 明确支持自定义书架和离线下载；Kobo 与 Kindle 强调跨设备保存位置、书签、标注和笔记。
- 起点读书强调书架、阅读进度、书签和笔记云同步；微信读书、起点读书和番茄小说都把讨论、笔记或社区反馈放在阅读之外的次级层。
- Apple Books 明确提供字体、背景主题、行高、字距、自动夜间主题、亮度与纵向滚动。

本文只提炼这些产品的通用交互，不复制任何品牌的配色、图标、排版或动效。对于 Story OS，商业产品中的书城、付费、广告、签到、阅读时长激励、排行榜和公开社区均不属于初始范围。

## 3. 设计原则

### 3.1 正文优先

阅读状态下只保留正文、轻量章节信息和必要的系统状态。工具栏默认隐藏或弱化，用户轻触正文中央后才显示。任何红点、弹窗、推荐卡和连续动画都不应打断阅读。

### 3.2 恢复优先

用户每次打开应用，都应能在一次点击内回到最近阅读位置。网络失败、版本更新、章节重排或字号变化时，系统要给出可理解的恢复结果，不能静默跳到错误位置。

### 3.3 渐进披露

书架展示最少信息；目录、搜索、设置、离线管理和反馈按需打开。高级状态（构建版本、可见性配置、缓存大小）只在相关页面出现。

### 3.4 权限前置

未授权数据不进入页面、搜索索引、离线包、Open Graph 元数据、错误日志或反馈附件。权限检查发生在生成和分发阶段，而不是依赖 CSS、路由或按钮隐藏。

### 3.5 舒适而非装饰

使用中性、低饱和度的阅读主题，避免高对比纯白大面积发光，也避免为了“纸张感”加入纹理噪点。动画短、少、可关闭；阴影和圆角只用于区分层级。

### 3.6 可访问的显式操作

手势可以加速，但不能代替按钮。目录、上一章、下一章、关闭弹层、返回书架都要有可见、可聚焦且有明确名称的控件。

## 4. 通用模式与 Story OS 取舍

| 能力 | 成熟阅读器的通用模式 | Story OS 推荐 | 不建议 |
|---|---|---|---|
| 书架 | 最近阅读优先；封面/标题/进度；列表或网格；离线标记 | 顶部固定“继续阅读”，下方为全部作品；支持最近阅读、标题和状态排序；卡片同时显示作品、当前章和进度 | 书城轮播、排行榜、签到、广告位 |
| 切书 | 返回书库后选择；保存每本书独立位置 | 阅读器左上返回书架；工具栏提供“最近阅读”快捷切换；切书前立即保存当前锚点 | 在正文边缘使用不可见手势切书 |
| 目录 | 全屏页或侧/底部抽屉；当前章节高亮；可快速跳转 | 手机使用全屏目录，平板可用侧栏；当前章自动滚入视野；显示已读/当前/未读状态；支持目录内筛选 | 仅显示章节编号、不显示当前定位 |
| 切章 | 章末上一章/下一章；目录跳转；部分产品支持滑动 | 连续滚动为默认；章末保留大按钮；工具栏另有上一章/下一章；横向滑动默认关闭，可在设置中启用 | 把屏幕左右点击区作为唯一导航 |
| 阅读进度 | 百分比、页码、章节剩余时间、跨设备同步 | 主要显示“第 N 章 · 本章 63%”；全书百分比作为次要信息；基于稳定锚点保存，百分比只用于展示 | 只保存像素滚动位置或全书百分比 |
| 字号/行距 | Aa 面板；实时预览；字体、字号、行高、边距、对齐 | 一个底部设置面板；字号、行距、段距、页边距分组；调整后保持语义位置；提供“恢复默认” | 设置变化后跳回章首；提供过多装饰字体 |
| 主题/夜间 | 浅色、暖色、深色；跟随系统；亮度独立控制 | 默认暖白、浅色、深色三套；支持跟随系统；深色主题避免纯黑白强对比；尊重系统减少动画 | 仅靠反色滤镜；正文与控件对比不足 |
| 章节导航 | 目录、进度条、Page Flip/缩略预览、章末导航 | v0.1 采用目录 + 章末按钮 + 工具栏；长篇可增加章节位置滑块，但滑块必须显示目标章节名并可取消 | 用精细滑块直接控制数千章，造成误触 |
| 搜索 | 书库搜索与书内搜索分开；结果含章节与上下文 | 书架搜索作品；阅读器搜索当前授权作品；结果展示章节名、短摘要和命中次数；点击后可返回原位置 | 搜索未授权章节；把查询发送到第三方分析服务 |
| 分享 | 分享书籍链接、引用或阅读卡片；系统分享面板 | 默认分享受控链接；分享前预览将发送的标题、章节和摘要；internal/editor 默认禁用公开分享；必要时只生成受访问控制链接 | 自动复制大段正文；在分享元数据中泄露后续章节 |
| 离线 | 用户主动下载；显示下载状态；自动保存阅读位置 | 明确“离线可用”开关、包大小、版本和更新时间；按作品下载；新版本完整后原子替换；退出账户清私密缓存 | 静默缓存所有作品；不同权限包混用缓存 |
| 反馈 | 笔记、标注、章评、讨论；失败时保留草稿 | 反馈入口在工具栏和章末；关联 chapter_id / anchor_id；本地草稿、明确发送状态、失败重试；反馈不能晋升 Canon | 在滚动中持续浮出评论；发送失败后丢内容 |
| 权限 | 账户书库、设备同步、下载和分享按需授权 | 构建可见性、托管访问控制、浏览器权限三层分离；v0.1 零敏感设备权限；最小化日志 | 依赖不可猜 URL；把令牌写进静态包；前端隐藏秘密 |
| 无障碍 | 系统字体缩放、读屏、对比度、键盘和大按钮支持 | 语义 HTML、可见焦点、屏幕阅读器章节播报、200% 缩放、320 CSS px 回流、至少 44 CSS px 舒适触控区 | 纯图标无名称；仅颜色表达状态；锁死缩放 |

## 5. 移动优先信息架构

阅读器只有两个主要空间：书架与正文。其他能力作为页面或临时层附着其上。

### 5.1 一级导航

v0.1 不需要常驻底部导航栏。只有书架是稳定首页，阅读页是沉浸任务页。设置、目录、搜索和离线管理从明确入口进入。

推荐路由概念如下，实际命名等待数据接口和框架确定：

- 访问门：未登录、链接失效、权限不足。
- 书架：当前可见性包内的全部作品与继续阅读。
- 作品概览：标题、简介、卷/章目录、离线状态。
- 阅读页：指定作品和章节。
- 书架搜索：按标题、作者/项目名、标签筛选。
- 书内搜索：只搜索当前作品的已授权内容。
- 离线管理：作品包版本、体积、更新时间与删除操作。

### 5.2 阅读页层级

从上到下建议为：

1. 安全区与轻量顶栏：返回书架、作品名、更多操作。
2. 章节标题与可选卷名。
3. 正文区域。
4. 章末状态：本章结束、阅读进度、上一章/下一章。
5. 唤起后出现的底部工具栏：目录、搜索、排版、反馈。

顶栏和底栏在滚动阅读时收起，轻触正文中央或向上轻滑后出现。读屏和键盘用户不依赖这一隐藏机制，所有操作仍在文档顺序中可达。

## 6. 页面与状态规格

### 6.1 访问门

目标：在任何内容加载前确定当前访问身份与内容包。

必须状态：

- 正在验证访问。
- 未登录或需要邮箱/一次性链接验证。
- 链接过期。
- 已登录但无此作品权限。
- 网络不可用，但本机存在仍有效的授权离线包。
- 网络不可用，且没有可用离线包。

错误页不能回显被拒绝作品的标题、章节名、摘要或封面。若离线包允许继续阅读，应明确显示“当前为离线副本”和最后验证时间。

### 6.2 书架

首屏结构：

- 页面标题与当前身份/可见性简短标识。
- “继续阅读”主卡：作品名、当前章、最近阅读时间、进度、继续按钮。
- 全部作品：紧凑列表为默认，封面网格为可选。
- 搜索、排序、离线管理入口。

卡片信息优先级：

1. 作品名。
2. 当前阅读章或“尚未开始”。
3. 阅读进度和离线状态。
4. 次要元数据，如系列顺序、更新状态。

必须状态：

- 首次进入且书架为空。
- 有作品但都未开始。
- 正常书架。
- 加载失败但有缓存。
- 加载失败且无缓存。
- 权限变化后部分作品被移除。
- 作品数据更新，阅读位置需要重新定位。

空状态应说明“当前访问配置没有可阅读作品”，不要误导为系统故障。

### 6.3 作品概览与目录

作品概览不是商业详情页，只承担开始阅读、继续阅读和定位章节：

- 作品名、可选作者/项目名、简短简介。
- 继续阅读或从头开始。
- 卷/章节目录。
- 离线下载/更新/删除。
- 当前构建版本和更新时间放在次级信息区。

目录行为：

- 默认定位并高亮当前章。
- 卷可折叠，但当前章所在卷自动展开。
- 章节行显示标题、可选序号、已读/当前状态。
- 数百或数千章时使用虚拟列表或分段加载。
- 支持按章节标题筛选。
- 未授权章节不会以锁形占位出现；它们根本不进入 reader/public 数据包。只有产品明确需要展示“存在但不可读”时，才由授权后的安全元数据包提供。

### 6.4 阅读页

默认行为：

- 纵向连续滚动。
- 进入后恢复到稳定文本锚点。
- 滚动停止后节流保存进度；切后台、切章、返回书架时立即保存。
- 浏览器返回键先关闭当前弹层，再返回上一页面。
- 正文可选择，除非分享策略明确禁止复制；禁止复制不是可靠的版权或保密措施，真正保护仍依赖分发范围。

阅读中状态：

- 正常在线。
- 正常离线。
- 当前章节有新版本但尚未切换。
- 文本已更新并自动定位到附近。
- 章节被删除、合并或权限撤销。
- 章节加载失败，可重试或返回目录。
- 下一章不存在，即作品/授权范围终点。

章节更新不能在用户正在阅读时突然替换 DOM 并改变位置。应在章末或工具栏提示“有新版本”，由用户主动刷新；安全撤权除外。

### 6.5 阅读设置

设置面板分为四组：

- 字体：系统中文无衬线、系统衬线两类即可；不默认下载远程字体。
- 字号：建议默认 18 CSS px，允许约 15–28 CSS px，并继续尊重浏览器/系统缩放。
- 间距：紧凑、舒适、宽松三档；默认行高约 1.8，中文段间距约 0.75em。
- 页面：窄/标准/宽边距；跟随系统、暖白、浅色、深色主题；连续滚动默认开启。

设置变化必须即时预览，关闭面板后保留。重新排版时使用“调整前视口中心附近的语义锚点”恢复，而不是复用旧 scrollTop。

主题建议：

| 主题 | 背景方向 | 正文方向 | 使用场景 |
|---|---|---|---|
| 暖白 | 低饱和米白 | 深灰褐 | 默认长时间阅读 |
| 浅色 | 柔和浅灰白 | 近黑 | 明亮环境 |
| 深色 | 深灰而非绝对黑 | 柔和灰白 | 夜间与低光环境 |

所有主题均需单独通过对比度测试，不能假设深色主题自动可访问。

### 6.6 搜索

书架搜索与书内搜索分开：

- 书架搜索只匹配作品级元数据。
- 书内搜索匹配当前作品的章节标题和授权正文。
- 搜索结果显示章节名、短上下文、命中数量。
- 点入结果后记录“搜索来源位置”，用户可一键返回结果列表或原阅读位置。
- 查询为空、无结果、索引未下载、索引版本过期均有独立状态。

搜索索引必须与内容包同一 profile、同一 build_id。任何降级路径都不能把查询发送到未批准的第三方服务。

### 6.7 分享预览

分享不是一个立即执行的图标按钮，而是两步流程：

1. 打开预览，明确显示将要分享的标题、章节、摘要和链接权限。
2. 用户确认后调用系统分享；不支持系统分享时再提供复制链接。

策略：

- public：可分享公开页面。
- reader：只分享该接收者可访问的受控链接，链接本身不应扩大权限。
- editor/internal：默认关闭对外分享；如确有需要，只生成受身份验证保护的链接。
- 默认不分享选中文本；未来若开放引用分享，必须设置长度上限、可见性检查和逐次确认。
- 页面标题、描述、社交预览图和错误监控字段都按 profile 生成，不能引用未来章节或内部设定。

### 6.8 离线管理

离线下载是显式动作，不是后台静默缓存全部正文。页面需显示：

- 作品名与当前离线状态。
- 预计大小、已下载大小。
- 内容版本、生成时间、最后联网验证时间。
- 下载、暂停/取消、更新、删除。
- 高敏感内容的离线禁用说明。

stateDiagram-v2
    [*] --> NotDownloaded
    NotDownloaded --> Downloading: 用户确认
    Downloading --> Ready: 校验完成
    Downloading --> Failed: 网络或空间不足
    Failed --> Downloading: 重试
    Ready --> Updating: 新版本且用户同意
    Updating --> Ready: 新包完整校验并切换
    Updating --> Ready: 更新失败，保留旧包
    Ready --> NotDownloaded: 用户删除或退出清理
~~~

新包必须完整下载并校验后再替换旧包，不能让目录来自版本 A、正文来自版本 B。退出登录或切换身份后，私密内容缓存必须清理；是否保留非敏感阅读进度，应由明确的本机隐私选项决定。

### 6.9 反馈

v0.2 的最小反馈表单：

- 类型：错字/行文、理解困难、节奏、连续性、情绪反馈、其他。
- 关联目标：novel_id、chapter_id、可选 anchor_id、content_revision。
- 用户输入：反馈正文。
- 可选严重程度或“阻碍继续阅读”标记。
- 身份、创建时间、发送状态和幂等键。

默认只记录锚点和版本，不把整段受限正文复制到外部反馈系统。若作者需要附带选文，应有单独策略和长度上限。

离线或发送失败时：

- 自动保留本地草稿。
- 显示“未发送”，不能显示成功态。
- 允许重试、编辑和删除。
- 服务器用幂等键防止重复提交。

反馈成功后只进入反馈层，状态可从 new 变为 triaged / resolved / dismissed；任何进入提案或 Canon 的动作都必须在作者工作流中另行发生。

## 7. 阅读位置模型

### 7.1 为什么不能只存百分比

字号、行距、屏幕宽度和章节修订都会改变页面高度。保存 scrollTop、页码或百分比会在重新排版后漂移。成熟阅读器的“跨设备续读”体验背后，本质上都需要与内容语义相关的位置。

### 7.2 建议的最小位置记录

字段名称仅为消费端需求示意，等待 Story OS 稳定数据接口后映射：

- novel_id
- chapter_id
- anchor_id：优先使用稳定段落/文本块 ID
- text_offset：锚点内部偏移
- context_hash：锚点附近短文本的摘要，用于修订后恢复
- percent_in_chapter：仅供展示和降级
- content_revision 或 content_hash
- updated_at
- completed

恢复顺序：

1. 同版本、同 anchor_id 精确恢复。
2. 同章节新版本中按 anchor_id 恢复。
3. 用 context_hash 或附近文本定位最接近位置。
4. 定位失败时回到章节开头，并提示“章节已更新，原位置无法精确恢复”。
5. 章节被合并/删除时，依据构建产物提供的替代映射定位；没有映射则回到目录，不猜测下一章。

### 7.3 保存时机

- 滚动停止约 500–1000ms 后节流保存。
- 页面隐藏、应用切后台、返回书架、切章前立即保存。
- 不在每个 scroll 事件同步写入存储。
- 多设备同步未来采用“更新时间 + 内容版本 + 最近位置历史”，冲突时展示可理解选择，不能简单以百分比最大者覆盖。

## 8. 视觉与排版建议

### 8.1 布局

- 手机正文左右内边距建议约 18 CSS px，窄屏最低不小于 14 CSS px。
- 平板和桌面正文设置最大行宽，避免横向铺满；建议约 38–44rem，再由真实中文样章验证。
- 使用 safe-area-inset 处理刘海、圆角和底部手势区。
- 使用动态视口单位时保留兼容回退，避免浏览器工具栏伸缩导致弹层跳动。
- 章节标题与正文之间有明显但克制的留白，不使用大幅封面作为阅读页背景。

### 8.2 字体

- 中文正文优先系统字体，避免远程字体增加闪烁、体积和隐私请求。
- 默认 18 CSS px、行高约 1.8 是测试起点，不是不可变标准。
- 不禁用浏览器缩放；用户在 200% 文本放大下仍可阅读和操作。
- 字重以常规为主，正文不要长期使用过细字重。
- 段首缩进遵循原稿/构建规则，不由前端同时再添加，避免双重缩进。

### 8.3 控件

- 舒适触控目标建议至少 44 × 44 CSS px；WCAG 2.2 AA 的最低目标为 24 × 24 CSS px 或具有足够间距。
- 图标按钮必须有可访问名称；常用关键操作优先“图标 + 文本”。
- 正文中的链接、脚注和反馈锚点要有足够间距，避免误触。
- 焦点样式在所有主题中清晰可见，不以去除 outline 作为视觉优化。

### 8.4 动效

- 工具栏和底部面板使用约 120–180ms 的淡入/位移即可。
- 跟随 prefers-reduced-motion 时移除非必要位移动画。
- 切章不做仿真翻页；内容加载可使用低干扰骨架或进度提示。

## 9. 技术建议

### 9.1 总体架构

~~~mermaid
flowchart LR
    A["已验证 Story OS 文件"] --> B["可见性裁剪与静态生成"]
    B --> C["profile-scoped 内容包"]
    B --> D["profile-scoped 搜索索引"]
    C --> E["静态 PWA"]
    D --> E
    E --> F["Cache Storage：应用壳与授权内容"]
    E --> G["IndexedDB：进度、设置、反馈草稿"]
    E --> H["受控反馈 API（v0.2）"]
    H --> I["feedback/inbox 或外部反馈层"]
~~~

建议采用 TypeScript 静态站技术栈，优先内容型、低 JavaScript 方案，例如 Astro 加少量交互岛；若团队最终选择其他支持静态输出的框架，也应保持以下边界：

- 正文和目录可在无复杂客户端状态管理时渲染。
- 阅读设置、进度、搜索、离线和反馈作为独立模块。
- 构建产物完全可由源文件重建。
- 不引入客户端数据库作为 Canon 或正文的事实源。

### 9.2 消费端数据边界

在正式接口稳定前，阅读器只提出最低消费需求，不决定 Schema：

- 构建清单：build_id、profile、生成时间、内容哈希、应用版本。
- 作品摘要：稳定 novel_id、标题、可选系列信息、简介、可见章节列表、更新状态。
- 章节：稳定 chapter_id、顺序、标题、结构化正文块、前后章引用、内容版本。
- 锚点：正文块稳定 ID；若源数据暂时没有，生成器必须产生可重复的确定性 ID，并记录局限。
- 搜索索引：只含当前 profile 可见内容，并绑定同一 build_id。
- 迁移映射：章节拆分、合并或重命名后的旧 ID 到新位置映射。

阅读器不能从文件路径、数组下标或显示标题推断稳定身份。

### 9.3 本地存储

- Cache Storage：应用壳、静态资源和用户明确下载的授权内容包。
- IndexedDB：阅读位置、显示设置、离线清单、待发送反馈草稿。
- localStorage：只用于极小的主题启动值，减少首屏主题闪烁；不保存正文、令牌或完整反馈。
- 所有本地记录都带 profile / user scope 和 build_id，避免身份切换串数据。

### 9.4 缓存策略

- 应用壳：版本化 cache-first，后台检查更新。
- 访问与清单：network-first，短超时后回退到仍有效缓存。
- 已下载正文：cache-first，但显示版本和离线状态。
- 未下载正文：network-first，不应因为访问过一次就无限期缓存全部私密内容。
- 更新：下载到新命名空间，校验成功后原子激活；失败保留上一稳定版本。
- 清理：退出、权限切换、作品撤权和用户主动删除时清除对应内容、索引与预览元数据。

必须在产品说明中写清：如果设备一直离线，已经下载的内容无法被服务器即时撤回。特别敏感的 internal 内容应默认不允许离线。

### 9.5 搜索

小到中型私人书库可在构建时生成轻量倒排索引，在设备本地搜索。中文索引需验证单字、双字切分与标点归一化，避免依赖空格分词。

大章节和多部长篇应：

- 按作品拆分索引。
- 只在进入作品或用户允许离线时下载。
- 结果只返回 chapter_id / anchor_id 与短摘要。
- 摘要在构建阶段生成并裁剪，不能从未授权全文动态截取。

### 9.6 分享

优先使用 Web Share API，在 HTTPS 和用户点击后调用；不支持时提供复制受控链接。分享能力不是读取通讯录权限，也不需要提前请求权限。

分享实现必须验证：

- 当前页面 profile 允许分享。
- 分享目标仍在授权构建物中。
- 标题、摘要、图片和 URL 均来自同一安全 profile。
- 复制/分享失败时给出可恢复提示。

### 9.7 反馈传输

v0.2 反馈接口建议只接受严格白名单字段，验证对象 ID、内容版本、身份和频率。需要：

- CSRF/来源检查或等价保护。
- 幂等键。
- 字数限制与速率限制。
- 失败重试和审计时间。
- 不在客户端暴露仓库写入令牌。
- 服务端只写反馈入口，不具备修改 Canon 的权限。

### 9.8 可观测性与隐私

默认只记录构建版本、错误类型和非内容性性能指标。日志不得包含：

- 正文、搜索词、选中文本或反馈正文。
- 完整分享令牌、邮箱、访问链接。
- 未授权作品名和章节名。

如果未来启用分析，应先定义最小事件清单，并允许私人部署关闭。

## 10. 权限模型

### 10.1 内容权限

| Profile | 内容范围 | 分享 | 离线 | 反馈 |
|---|---|---|---|---|
| internal | 作者完整资料与正文 | 默认禁止公开分享 | 默认关闭，高敏感内容禁止 | 作者自用，可关联全部对象 |
| editor | 指定正文与设定 | 仅身份保护链接 | 按授权策略 | 可提交编辑反馈 |
| reader | 指定章节、无未来剧透 | 受控链接 | 可选下载授权章节 | 可提交读者反馈 |
| public | 明确批准的公开内容 | 公开链接 | 可选 | 默认关闭或受限 |

独立 profile 应生成独立内容包和搜索索引。不要把 internal 全量数据下发后再用角色字段过滤。

### 10.2 浏览器/设备权限

v0.1 推荐：

- 不请求位置、相机、麦克风、通讯录、蓝牙、通知。
- 系统分享只在用户点击后调用。
- 剪贴板写入只作为明确“复制链接”操作。
- 持久存储请求只在用户选择离线下载后考虑，并解释用途。
- 屏幕常亮属于可选增强，默认不启用；拒绝权限不影响阅读。

### 10.3 托管访问控制

- public 才适合无身份的公开静态托管。
- reader/editor/internal 需要平台级身份或访问控制，不能只使用 robots.txt、不可猜路径或前端密码。
- 私密内容的 CDN、错误页、站点地图、RSS、社交预览和搜索引擎索引都要关闭或裁剪。
- 访问令牌不进入 Git、静态 JS、页面源码或分享截图。

## 11. 无障碍基线

目标至少对齐 WCAG 2.2 AA，并在真实移动读屏上验证。

### 11.1 语义与读屏

- 页面有唯一、准确标题；正文使用 main、article、header、nav 等语义结构。
- 章节标题为真实 heading，不用只靠字号模拟。
- 目录是可导航列表，当前章使用 aria-current 或等价语义。
- 章节切换后更新页面标题，并把焦点/读屏位置放到新章节标题，避免从浏览器顶部重新读。
- 离线完成、反馈发送成功/失败等异步结果使用非打断式状态播报。
- 图标按钮有明确名称，例如“打开目录”“下一章”，不能只读出“按钮”。

### 11.2 键盘与焦点

- 所有操作可用键盘完成。
- 弹层打开后焦点进入弹层；关闭后回到触发按钮。
- 焦点顺序与视觉顺序一致。
- 所有主题都有明显焦点环。
- 提供“跳到正文”链接。

### 11.3 视觉

- 普通正文与背景对比度至少 4.5:1；大文本至少 3:1。
- 320 CSS px 宽度下无需双向滚动，目录、设置和反馈表单不溢出。
- 用户把行高调到 1.5 倍、段后距 2 倍、字距 0.12 倍、词距 0.16 倍时，不丢内容或功能；中文脚本不使用的属性按 WCAG 例外处理。
- 状态不能只靠颜色表达；当前章还需文字/图标/语义。
- 支持浏览器 200% 缩放和系统大字体，不锁 maximum-scale。

### 11.4 运动与触控

- 尊重 prefers-reduced-motion。
- 不强制复杂滑动、拖拽或多点手势。
- 触控目标以至少 44 CSS px 为舒适目标，并满足 WCAG 2.2 最低尺寸/间距要求。
- 点击正文显示工具栏时，不阻止文本选择、读屏探索或浏览器原生返回手势。

建议人工测试组合：

- iPhone Safari + VoiceOver。
- Android Chrome + TalkBack。
- 桌面 Chrome/Edge + 键盘。
- 200% 缩放、320 CSS px 视口、深色主题、减少动画。

### 12.1 建议设备覆盖

- iOS Safari：当前主版本与前一主版本。
- Android Chrome：当前稳定版，中端设备与低内存模式。
- 桌面 Chrome/Edge：键盘与响应式回退。
- 可选 Firefox：基础阅读和无障碍回归。
- 安装态 PWA与普通浏览器标签页都要测试。

### 12.2 自动化边界

适合自动化：

- 可见性包、索引和分享元数据泄露检查。
- 路由、目录、切章、进度、设置与离线状态机。
- axe 等静态无障碍扫描。
- 缓存版本完整性和反馈幂等。

必须人工验证：

- 中文长文真实阅读舒适度。
- VoiceOver/TalkBack 的章节切换和弹层体验。
- 深色主题眩光、字号和行距档位。
- iOS 安装态 PWA 的返回、分享和离线行为。
- 疲惫状态下是否能在一次点击内继续阅读。

## 13. 分阶段范围

### v0.1：只读 PWA

必须：

- 访问门与 profile-scoped 静态内容包。
- 书架、继续阅读、切书。
- 作品目录、切章。
- 稳定锚点进度与恢复。
- 字号、行距、边距、三套主题、跟随系统。
- 书架搜索和授权书内搜索。
- 受控链接分享。
- 显式离线下载和版本管理。
- WCAG 2.2 AA 基线与真实读屏测试。

可以后置：

- 横向仿真翻页。
- 阅读目标、时长统计和连续打卡。
- 听书/TTS。
- 高亮、笔记和复杂标注。
- 关系图、时间线等 Story OS 专业视图。

### v0.2：反馈

- 章级和锚点级反馈。
- 本地草稿、失败重试、幂等提交。
- editor / reader 身份。
- 作者侧反馈状态流转。
- 反馈不得直接修改 Canon。

### 暂不进入近期范围

- 公开社区和实时章评。
- 在线正文编辑与多人协作。
- 自动接受反馈。
- 广告、付费、推荐、积分、榜单。
- 原生 iOS/Android 应用。

## 14. 实现前必须由主数据流确认的接口问题

阅读器实现应等待以下接口稳定，不应在 UI 工程中自行发明答案：

1. novel_id、chapter_id 和正文块 anchor_id 的来源与稳定性。
2. 章节排序、卷结构、拆分/合并迁移映射。
3. profile 构建产物的清单格式和版本标识。
4. 章节正文是预渲染安全 HTML、结构化块，还是 Markdown AST。
5. 可搜索字段、摘要生成与中文分词策略。
6. 作品/章节的离线许可和分享许可字段。
7. 阅读进度是否仅本机保存，还是存在受控同步端点。
8. 反馈入口的身份、对象引用、写入位置和幂等协议。
9. 托管平台如何验证 reader/editor/internal 身份。
10. 撤权、过期和登出时的缓存清理信号。

接口确定后，先用一部小型真实作品和一个 reader profile 做纵向切片，不同时实现关系图、时间线、反馈和复杂标注。

## 15. v0.1 验收标准

- 手机打开后，一次点击可从书架恢复最近阅读位置。
- 两部作品来回切换，各自位置稳定且不串读。
- 调整字号、行距、主题或旋转屏幕后，仍停留在同一句附近。
- 目录能快速定位当前章，章末能明确进入下一章。
- 离线包下载、更新失败和删除均有可理解状态；更新失败保留旧版本。
- reader/public 构建物、搜索索引、缓存和分享元数据中不存在更高可见性内容。
- 键盘、VoiceOver、TalkBack、200% 缩放和减少动画均能完成核心阅读路径。
- 网络失败、章节修订、链接过期、空间不足和反馈失败都不会静默丢失进度或草稿。
- 整体视觉不使用任何调研品牌的特征性配色、图标、纹理或动效。
